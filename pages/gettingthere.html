Overview

A social app to help NYC commuters communicate with other members to get the latest information on train delays and etas.

Technology Used

Django | PostgreSQL | MVC | Python | MTA API | HTML5 | Materialize | JavaScript

Features

  • This app was build as a team of three for New York commuters that needed and could share first hand train information.
  • Users are able to access alerts for any train station and discuss their daily commute.
  • The front end was made using Django with Python logic.
  • Templates did involve JavaScript, but more so for enabling Materialize features.
  • The app was developed following the MVC model.
  • The data for train stations was obtained through the MTA API.
